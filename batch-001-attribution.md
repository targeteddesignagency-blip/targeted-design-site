## 4. PHONE TRACKING STRATEGY

### Current State
- **Single number:** (210) 903-5551 — rings to Voice Agent (bilingual, 24/7)
- **Problem:** Cannot differentiate which ZIP drove the call when all 4 ZIPs share one number
- **Impact:** Without ZIP-level call attribution, we can't calculate per-ZIP ROI or optimize drop order for Batch 002

### Option A: CallRail — Recommended for Scale

| Aspect            | Detail                                                                      |
|-------------------|-----------------------------------------------------------------------------|
| **What**          | Purchase 4 local (210) tracking numbers, one per ZIP                        |
| **Cost**          | ~$95/mo (4 numbers × $3/mo + $45 base + usage ~$0.05/min)                 |
| **Setup time**    | 1–2 business days                                                           |
| **How it works**  | Each ZIP's card prints a unique tracking number. CallRail forwards all calls to (210) 903-5551 (Voice Agent). CallRail logs: caller ID, call duration, recording, which tracking number was dialed. |
| **Integration**   | CallRail → Google Analytics (call events), Google Sheets (via Zapier), Webhooks to CRM |
| **Pros**          | Full analytics dashboard, call recording, keyword-level attribution, IVR, SMS follow-up, GA4 integration out of the box |
| **Cons**          | Monthly cost, need to provision numbers before print                        |
| **Decision**      | ✅ **RECOMMENDED** — best attribution fidelity, scalable across future batches |

### Option B: Twilio Sub-Numbers — Budget Alternative

| Aspect            | Detail                                                                      |
|-------------------|-----------------------------------------------------------------------------|
| **What**          | Purchase 4 Twilio local numbers, forward to (210) 903-5551                 |
| **Cost**          | ~$4.40/mo (4 × $1.10/mo) + $0.0085/min inbound — roughly $10-15/mo total   |
| **Setup time**    | Same day (programmatic via Twilio Console or API)                          |
| **How it works**  | Twilio number → TwiML `<Dial>` → Voice Agent number. Log each call with `From`, `To` (tracking number), `CallSid` to Google Sheet via webhook. |
| **Integration**   | Custom webhook (Twilio → Google Sheets), no built-in analytics              |
| **Pros**          | Extremely cheap, fast setup, programmatic control, pairs with existing Twilio SMS infrastructure |
| **Cons**          | No analytics dashboard, no call recording out of the box (needs extra code), manual Google Sheet logging, no GA4 integration without custom work |
| **Decision**      | ✅ Acceptable if budget constrained — requires building a logging webhook   |

### Option C: Manual Google Sheet Logging — Zero Cost, High Labor

| Aspect            | Detail                                                                      |
|-------------------|-----------------------------------------------------------------------------|
| **What**          | All calls go to (210) 903-5551. Voice Agent asks "What ZIP code is your business in?" at qualification. Manual or automated entry into Google Sheet. |
| **Cost**          | $0                                                                          |
| **Setup time**    | None — update Voice Agent script                                            |
| **How it works**  | Voice Agent captures ZIP during qualification call → logs to Google Sheet. Caller self-reports ZIP. |
| **Integration**   | Voice Agent → Google Sheet (already in pipeline)                            |
| **Pros**          | Zero cost, no new numbers, leverages existing Voice Agent + Google Sheet flow |
| **Cons**          | **Attribution is unreliable** — caller may not know ZIP, may give wrong ZIP, or may hang up before asked. No way to attribute missed/abandoned calls. |
| **Decision**      | ⚠️ **LAST RESORT ONLY** — supplement but don't rely on this for attribution |

### Phone Tracking Decision Matrix

| Criteria              | CallRail        | Twilio Sub-Nums  | Manual Sheet    |
|-----------------------|-----------------|-------------------|-----------------|
| ZIP-level attribution | 100% accurate   | 100% accurate     | ~60% accurate   |
| Setup speed           | 1–2 days        | Same day          | None            |
| Monthly cost          | ~$95            | ~$15              | $0              |
| Call recording         | ✅ Built-in     | ❌ Requires code  | ❌ No           |
| GA4 integration       | ✅ Built-in     | ❌ Custom webhook | ❌ No           |
| Scalable to Batch 002+| ✅ Add numbers  | ✅ Add numbers    | ⚠️ Unreliable   |
| Pairs with Twilio SMS | ✅              | ✅ (same ecosystem) | ✅           |

### Recommendation

**PRIMARY: CallRail** — Provision 4 local (210) numbers before print. Full attribution, full analytics, GA4 integration. The $95/mo pays for itself with one additional Standard client ($550/mo) attributed to the right ZIP.

**FALLBACK: Twilio sub-numbers** — If CallRail provisioning is delayed or budget is constrained, use Twilio sub-numbers with a logging webhook. Cheaper but requires development.

**SUPPLEMENT: Manual sheet logging** — Always run this as a cross-check. Voice Agent should capture ZIP anyway — it validates CallRail/Twilio data and catches caller self-reported location.

## 7. ATTRIBUTION FLOW — END TO END

```
EDDM CARD (per ZIP)
    │
    ├── QR CODE SCAN ──→ targeted-design.com?utm_source=eddm&utm_medium=direct-mail&utm_campaign=batch-001&utm_content={zip}
    │                          │
    │                          ├── GA4 session starts → utm_source=eddm, medium=direct-mail, campaign=batch-001
    │                          ├── Custom dim: zip_code = {zip}
    │                          └── User action → form submit / phone click → conversion event
    │
    └── PHONE CALL ──→ (210) 903-5551 [or ZIP-specific tracking number]
                           │
                           ├── Voice Agent answers (bilingual)
                           ├── Qualifies lead → asks industry, business name
                           ├── Logs to Google Sheet: timestamp, caller ID, ZIP (from tracking number OR self-report), industry
                           └── Books consultation → Google Calendar → SMS confirmation (Twilio)
```

### Data Reconciliation (Daily)

Cross-reference these sources to calculate true per-ZIP response rate:

| Source           | Metric                              | Join Key          |
|------------------|-------------------------------------|-------------------|
| GA4              | Sessions by utm_content             | ZIP code          |
| CallRail/Twilio  | Calls by tracking number             | Tracking number → ZIP |
| Google Sheet     | Voice Agent qualification log       | Caller ID + ZIP   |
| Square           | Payments / subscriptions             | Business name     |

**Per-ZIP ROI = (Revenue from ZIP attributions) − (Print + Postage cost for ZIP's routes)**
